# 用户操作手册 - 汉字拼音标注工具

## 1. 产品概述

汉字拼音标注工具是一个专业的中文文本处理软件，能够将中文文本转换为带拼音标注的格式，并支持导出为DOCX和PDF文件。本工具特别适合教育工作者、学生、语言学习者使用。

### 1.1 主要功能
- **汉字转拼音**: 自动将中文文本转换为拼音标注
- **多格式导出**: 支持DOCX和PDF两种格式
- **样式自定义**: 可调整字体、字号、行距等参数
- **智能布局**: 自动优化每行显示字数
- **端口自动检测**: 智能选择可用端口
- **浏览器自动打开**: 一键启动，自动打开浏览器

### 1.2 系统要求
- 操作系统: Windows 7/8/10/11, macOS, Linux
- 浏览器: Chrome, Firefox, Edge, Safari
- 网络: 本地网络连接
- 权限: 管理员权限（端口80需要）

## 2. 安装与启动

### 2.1 安装步骤

#### 2.1.1 环境准备
1. 下载并安装 Node.js（版本14.0.0或更高）
   - 访问 [nodejs.org](https://nodejs.org/)
   - 下载LTS版本
   - 按提示完成安装

2. 验证安装
   ```bash
   node --version
   npm --version
   ```

#### 2.1.2 项目安装
1. 获取项目文件
   - 从GitHub下载ZIP文件，或
   - 使用git克隆: `git clone https://github.com/trae-ai/word-pinyin.git`

2. 进入项目目录
   ```bash
   cd word-pinyin
   ```

3. 安装依赖
   ```bash
   npm install
   ```

### 2.2 启动服务

#### 2.2.1 快速启动
```bash
npm start
```

#### 2.2.2 开发模式启动
```bash
npm run dev
```

#### 2.2.3 端口说明
- **首选端口80**: 标准HTTP端口
- **备选端口8080**: 常用备选端口  
- **自动端口**: 如果以上端口被占用，自动选择随机端口

### 2.3 访问应用
启动成功后，浏览器会自动打开应用界面。如未自动打开，请手动访问：
- http://localhost（使用端口80时）
- http://localhost:8080（使用端口8080时）
- http://localhost:XXXX（使用随机端口时，XXXX为实际端口号）

### 3.2 功能区域说明

#### 3.2.1 文本输入区域
- **位置**: 界面上方大文本框
- **功能**: 输入需要转换的中文文本
- **提示**: 支持长文本，建议分段输入
- **限制**: 无字符数限制，但过长文本可能影响性能

#### 3.2.2 转换选项区域
- **拼音样式下拉框**: 
  - 声调符号: 使用āáǎà等符号标注声调
  - 数字标注: 使用a1a2a3a4等数字标注声调
- **大小写下拉框**:
  - 小写: 拼音字母使用小写
  - 大写: 拼音字母使用大写
- **ü字符下拉框**:
  - v: 使用v代替ü（如lv）
  - u: 使用u:表示ü（如lu:）
  - ü: 保持原字符ü

#### 3.2.3 每行字数控制
- **输入框**: 设置每行显示的中文字符数
- **范围**: 5-50个字符
- **建议**: 20个字符适合A4纸张
- **调整**: 使用上下箭头或直接输入数字

#### 3.2.4 操作按钮
- **转换拼音按钮**: 执行拼音转换，显示结果
- **导出DOCX按钮**: 将结果导出为Word文档
- **导出PDF按钮**: 将结果导出为PDF文档

#### 3.2.5 结果显示区域
- **位置**: 界面下方大文本框
- **功能**: 显示转换后的拼音标注结果
- **格式**: 显示为表格形式，包含拼音和对应汉字

## 4. 操作流程

### 4.1 基本转换流程

#### 4.1.1 步骤说明
1. **输入文本**
   - 在文本输入区域输入中文内容
   - 示例: "汉字拼音标注工具使用说明"

2. **设置选项**
   - 选择拼音样式（推荐"声调符号"）
   - 选择大小写（推荐"小写"）
   - 选择ü字符处理（推荐"v"）
   - 设置每行字数（推荐"20"）

3. **执行转换**
   - 点击"转换拼音"按钮
   - 等待转换完成（通常1-3秒）

4. **查看结果**
   - 在结果显示区域查看转换效果
   - 检查拼音标注是否正确

#### 4.1.2 转换示例
**输入文本**: "汉字拼音标注工具使用说明"

**转换结果**:
```
hàn zì  pīn yīn  biāo zhù  gōng jù  shǐ yòng  shuō míng
汉 字   拼 音   标 注    工 具   使 用   说 明
```

### 4.2 文件导出流程

#### 4.2.1 DOCX导出
1. 完成拼音转换（必须先转换才能导出）
2. 点击"导出DOCX"按钮
3. 选择保存位置（浏览器自动下载）
4. 打开文件检查内容

#### 4.2.2 PDF导出
1. 完成拼音转换（必须先转换才能导出）
2. 点击"导出PDF"按钮  
3. 选择保存位置（浏览器自动下载）
4. 使用PDF阅读器打开查看

### 4.3 高级功能使用

#### 4.3.1 长文本处理
对于长文本（超过1000字符），建议：
1. 分段输入和处理
2. 保持每段格式一致
3. 分别导出后合并

#### 4.3.2 特殊字符处理
- **标点符号**: 自动保留在正确位置
- **数字**: 保持原样，不转换
- **英文**: 保持原样，不转换
- **空格**: 自动处理，保持格式

## 5. 故障排除

### 5.1 常见问题及解决方案

#### 5.1.1 转换失败
**现象**: 点击"转换拼音"后无反应或报错
**原因**: 
- 输入文本为空
- 网络连接问题
- 服务器处理错误

**解决**:
1. 检查输入文本是否有效
2. 检查网络连接
3. 重新启动服务
4. 查看错误日志

#### 5.1.2 文件下载失败
**现象**: 点击导出按钮后文件无法下载
**原因**:
- 浏览器下载设置问题
- 临时文件目录权限问题
- 文件名编码问题

**解决**:
1. 检查浏览器下载设置
2. 检查temp目录权限
3. 尝试使用不同浏览器
4. 清理浏览器缓存

#### 5.1.3 DOCX文件打开错误
**现象**: Word打开文件时报错或显示异常
**原因**:
- Word版本兼容性问题
- 文件下载不完整
- 字体不支持

**解决**:
1. 确保使用Word 2010或更新版本
2. 重新下载文件
3. 安装中文字体
4. 尝试使用WPS打开

#### 5.1.4 PDF中文显示为方框
**现象**: PDF文件中中文显示为方框或乱码
**原因**:
- PDF阅读器不支持中文
- 系统缺少中文字体
- 字体嵌入失败

**解决**:
1. 更新PDF阅读器
2. 安装中文字体包
3. 使用Adobe Reader打开
4. 重新生成PDF

### 5.2 性能优化建议

#### 5.2.1 大文本处理
- **分批处理**: 将大文本分成小段处理
- **增加内存**: 为Node.js分配更多内存
- **使用SSD**: 提高文件读写速度

#### 5.2.2 批量处理
- **脚本自动化**: 编写脚本批量处理文件
- **并行处理**: 同时处理多个请求
