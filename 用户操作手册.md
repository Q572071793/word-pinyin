# 中文拼音转换工具用户操作手册

## 1. 产品简介

中文拼音转换工具是一款专为教育工作者设计的实用软件，能够将中文文本快速转换为带拼音标注的格式。支持实时转换、DOCX文件处理、多种导出格式等功能，是教师制作拼音教学材料的得力助手。

## 2. 系统要求

### 2.1 运行环境
- **操作系统**: Windows 7/8/10/11, Linux, macOS
- **Node.js**: 版本 14.0 或更高
- **内存**: 至少 2GB RAM
- **磁盘空间**: 至少 100MB 可用空间

### 2.2 浏览器支持
- Chrome 80+
- Firefox 75+
- Safari 13+
- Edge 80+

## 3. 快速开始

### 3.1 启动服务

#### 方法1：使用启动脚本（推荐）
```bash
# Windows系统
双击运行 start.bat

# Linux/Mac系统
./start-server.sh
```

#### 方法2：手动启动
```bash
node start-server.js
```

启动成功后，系统会自动打开浏览器并访问应用地址（通常是 http://localhost:3000）。

## 4. 功能使用

### 4.1 实时拼音转换

#### 操作步骤
1. 在输入框中输入中文文本
2. 系统会自动开始转换（无需点击按钮）
3. 转换结果会实时显示在下方区域

#### 示例
**输入**: `你好世界`
**输出**: 
```
nǐ  hǎo  shì  jiè
你  好   世   界
```

#### 注意事项
- 转换过程有轻微延迟是正常的（约0.5秒）
- 输入过程中会看到"正在处理..."的提示
- 支持连续输入，系统会自动处理

### 4.2 文件上传转换

#### 支持的文件格式
- DOCX格式（.docx）
- 文件大小限制：10MB

#### 操作步骤
1. 点击【上传DOCX】按钮
2. 选择要转换的DOCX文件
3. 等待系统处理完成
4. 查看转换结果

#### 拖拽上传（便捷方式）
1. 打开文件管理器
2. 选中DOCX文件
3. 拖拽文件到应用界面
4. 系统自动开始处理

#### 处理结果说明
转换完成后，您会看到：
- 转换后的带拼音文本
- 原文本内容（便于对比）
- 段落数量统计
- 中文字符统计

### 4.3 结果导出

#### 导出为DOCX文件
1. 完成文本转换后
2. 点击【导出DOCX】按钮
3. 选择保存位置
4. 文件将自动下载

#### 导出为PDF文件
1. 完成文本转换后
2. 点击【导出PDF】按钮
3. 选择保存位置
4. 文件将自动下载

#### 打印功能
1. 在浏览器中按 Ctrl+P（Windows）或 Cmd+P（Mac）
2. 选择打印设置
3. 可以直接打印当前显示的内容

## 5. 高级功能

### 5.1 符号位置优化

系统自动优化特殊符号的位置：
- **右下角对齐**: "，。！？；："等标点符号
- **中间对齐**: "·"间隔号
- **底部对齐**: 其他标点符号

### 5.2 智能换行

系统支持两种换行方式：
- **自动换行**: 当一行文字过长时自动换行
- **手动换行**: 检测到回车符时强制换行

### 5.3 段落分析

上传DOCX文件后，系统会提供：
- 段落数量统计
- 每个段落的中文字符数
- 总中文字符数统计

## 6. 常见问题

### 6.1 转换相关问题

**Q: 为什么有些汉字没有拼音？**
A: 请确认输入的是标准中文字符。系统只处理常用汉字，特殊字符或符号不会转换。

**Q: 多音字如何处理？**
A: 系统会自动选择最常见的读音。目前不支持手动选择多音字的不同读音。

**Q: 转换速度很慢怎么办？**
A: 如果文本过长（超过5000字符），建议分段处理。系统正在优化大文本处理性能。

### 6.2 文件处理问题

**Q: 上传DOCX文件失败？**
A: 请检查：
- 文件是否为标准DOCX格式（非DOC格式）
- 文件是否损坏
- 文件大小是否超过10MB限制

**Q: 导出的文件格式不对？**
A: 请确认：
- 浏览器是否支持文件下载
- 是否有足够的磁盘空间
- 文件名是否包含特殊字符

### 6.3 显示问题

**Q: 拼音显示错位？**
A: 请尝试：
- 刷新页面重新加载
- 检查浏览器是否为最新版本
- 清除浏览器缓存

**Q: 界面显示异常？**
A: 建议使用Chrome或Firefox浏览器的最新版本。IE浏览器可能不支持某些功能。

## 7. 故障排除

### 7.1 服务启动失败

**症状**: 启动脚本运行后没有反应
**解决方法**:
1. 检查Node.js是否已安装：`node --version`
2. 检查端口3000是否被占用
3. 查看logs目录下的错误日志
4. 尝试手动启动：`node start-server.js`

### 7.2 浏览器无法访问

**症状**: 浏览器显示"无法连接"
**解决方法**:
1. 确认服务已成功启动
2. 检查防火墙设置
3. 尝试访问：http://localhost:3000
4. 检查是否有其他程序占用端口

### 7.3 转换功能异常

**症状**: 转换按钮点击无反应
**解决方法**:
1. 打开浏览器开发者工具（F12）
2. 查看控制台是否有错误信息
3. 检查网络请求是否成功
4. 查看logs目录下的应用日志

## 8. 性能优化建议

### 8.1 大文本处理
- 建议单次处理不超过5000字符
- 大文档建议分段处理
- 使用文件上传功能处理大文本

### 8.2 浏览器优化
- 使用Chrome或Firefox最新版本
- 定期清理浏览器缓存
- 关闭不必要的浏览器扩展

### 8.3 系统优化
- 确保有足够的可用内存
- 定期清理临时文件
- 监控系统日志及时发现异常

## 9. 安全注意事项

### 9.1 文件安全
- 只上传信任的DOCX文件

### 9.2 网络安全
